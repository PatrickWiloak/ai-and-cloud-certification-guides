# Microsoft Copilot and Agent Administration Fundamentals (AB-900)

## Exam Overview

The Copilot and Agent Administration Fundamentals certification validates foundational knowledge of Microsoft 365 Copilot, Microsoft Copilot Studio, AI agents, and the governance and security features that support them. This exam is intended for candidates who want to demonstrate their understanding of how AI-powered copilots and agents work within the Microsoft ecosystem, including IT administrators, business users, and anyone involved in evaluating or deploying Copilot solutions.

**Exam Details:**
- **Exam Code:** AB-900
- **Full Name:** Microsoft Certified: Copilot and Agent Administration Fundamentals
- **Duration:** 45 minutes
- **Number of Questions:** ~35 questions
- **Passing Score:** 700 out of 1000
- **Question Types:** Multiple choice, multiple response, drag-and-drop
- **Cost:** $99 USD (free retake within 30 days)
- **Prerequisites:** None
- **Validity:** Does not expire (fundamentals level)
- **Delivery:** Pearson VUE or Certiport

## Exam Domains

### 1. Describe Microsoft 365 Copilot (25-30%)
- **Copilot Features and Capabilities**
  - Copilot integration across M365 apps (Word, Excel, PowerPoint, Outlook, Teams)
  - Copilot Chat experience and cross-app capabilities
  - Content creation, summarization, and data analysis features
- **Copilot Architecture**
  - Role of large language models (LLMs), Microsoft Graph, and Semantic Index
  - How Copilot processes prompts and returns responses
- **Licensing and Deployment**
  - Licensing prerequisites and add-on model
  - Deployment steps and admin center configuration
  - User enablement and adoption planning
- **Use Cases**
  - Productivity scenarios across different roles and departments
  - When Copilot is appropriate vs other tools

### 2. Describe Microsoft Copilot Studio (25-30%)
- **Building Copilots**
  - Low-code authoring environment
  - Creating and managing custom copilots
  - Generative AI capabilities (generative answers, generative actions)
- **Topics and Conversation Design**
  - Trigger phrases, conversation nodes, and branching logic
  - System topics and fallback handling
- **Actions and Connectors**
  - Power Automate flow integration
  - Power Platform connectors and custom connectors
  - Dataverse data access and integration
- **Publishing and Analytics**
  - Multi-channel deployment (Teams, web, Power Apps)
  - Performance monitoring and conversation analytics
  - Customer satisfaction tracking

### 3. Describe AI Agents (20-25%)
- **Agent Concepts**
  - Definition and characteristics of AI agents
  - Difference between copilots and agents
  - Autonomy spectrum (guided to fully autonomous)
- **Types of Agents**
  - Conversational agents, task-based agents, autonomous agents
  - Declarative agents for Microsoft 365 Copilot
  - Agents in Microsoft Teams
- **Agent Orchestration**
  - Orchestration patterns (sequential, parallel, hierarchical, event-driven)
  - Multi-agent coordination and management
  - Human oversight and intervention

### 4. Describe Governance and Security for Copilot (20-25%)
- **Data Protection**
  - How Copilot handles organizational data
  - Permission inheritance and oversharing risks
  - Data residency and processing boundaries
- **Compliance**
  - Microsoft Purview integration (sensitivity labels, DLP, audit logs)
  - eDiscovery and retention for Copilot content
  - Communication compliance monitoring
- **Identity and Access**
  - Microsoft Entra ID integration
  - Conditional Access policies for Copilot
  - Role-based access control (RBAC)
- **Responsible AI**
  - Microsoft's responsible AI principles applied to Copilot
  - Content safety and filtering
  - Transparency and accountability
- **Admin Controls**
  - Tenant-level Copilot settings management
  - Plugin and extension governance
  - Copilot Studio environment management

### Hands-on Practice
- Explore Microsoft 365 Copilot features (trial or production environment)
- Build a basic copilot in Copilot Studio using the free trial
- Navigate the Microsoft 365 admin center Copilot settings
- Review Microsoft Purview compliance features
- Explore Copilot Studio analytics and publishing options

### Exam Strategy
- This is a fundamentals exam: focus on breadth rather than depth
- Understand "what" and "when" rather than "how to implement"
- Know the differences between Copilot, Copilot Studio, and agents
- Understand governance and compliance features at a conceptual level
- Be familiar with licensing requirements and deployment prerequisites

### Common Gotchas
- Copilot respects existing Microsoft 365 permissions (it does not bypass access controls)
- Copilot data is not used to train foundation models
- Know the distinction between copilots (human-in-the-loop) and agents (more autonomous)
- Understand that Copilot Studio is a low-code platform, not a developer-only tool
- Sensitivity labels from source documents carry through to Copilot-generated content
- Copilot is an add-on license, not included in base M365 subscriptions
