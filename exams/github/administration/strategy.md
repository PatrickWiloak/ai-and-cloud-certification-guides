# GitHub Administration Study Strategy

## Study Approach

### Phase 1: Identity and Access (Weeks 1-2)
1. **Enterprise Fundamentals** - GHEC vs GHES, support, licensing
2. **SAML SSO** - Configuration, IdP integration, recovery codes
3. **SCIM Provisioning** - Automated user management, team sync
4. **EMU** - Enterprise Managed Users vs standard accounts
5. **Roles and Permissions** - Org roles, enterprise roles, custom roles

### Phase 2: Repositories and Actions (Weeks 2-3)
1. **Repository Management** - Visibility, templates, policies
2. **Branch Protection** - Rules vs rulesets, required reviews, status checks
3. **Actions Policies** - Allowed actions, default permissions
4. **Self-Hosted Runners** - Groups, labels, security, scaling
5. **Secrets Management** - Org secrets, environment secrets

### Phase 3: Security and Enterprise (Weeks 3-4)
1. **GHAS Administration** - Licensing, enablement, code scanning, secret scanning
2. **Enterprise Policies** - Repository creation, base permissions, fork policies
3. **Audit Logging** - What is logged, API access, log streaming
4. **IP Allow Lists** - Network access control
5. **Practice exams and review**

### Readiness Indicators
- [ ] You can explain SAML SSO and SCIM and when to use each
- [ ] You understand EMU vs standard organization accounts
- [ ] You know Actions policies and runner management
- [ ] You can describe GHAS features and administration
- [ ] You understand audit logging and enterprise policies
- [ ] You score 75%+ consistently on practice exams
